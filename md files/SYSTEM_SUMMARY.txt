================================================================================
                        BOOKING WORKFLOW
================================================================================

STEP 1: STUDENT INITIATES
 Creates booking request

STEP 2: SYSTEM CREATES RECORD
   Booking created with status: "pending"
   Total amount calculated based on room price

STEP 3: MANAGER REVIEWS
   Manager sees booking in dashboard (/manager/bookings)
   Reviews student details and booking period

STEP 4: MANAGER APPROVES/REJECTS
   IF APPROVED:
 "approved"
   - Bed marked as occupied
   - Student notified
   
   IF REJECTED:
 "rejected"
   - Bed remains available
   - Student can try another room

STEP 5: STUDENT OCCUPIES/CANCELS
   Student checks in on check_in_date
   Can cancel if status is pending/approved
   
   OR at check_out_date:
 "completed"
   - Bed marked as unoccupied
   - Student can book again

PAYMENT FLOW:
 Payment record with "pending" status
   2. Student initiates payment
   3. System updates payment status (paid/failed)
   4. Can refund if needed
